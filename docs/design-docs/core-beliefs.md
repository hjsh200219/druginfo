# Core Beliefs - DrugInfo MCP

## 1. MCP 프로토콜 우선

의약품 정보를 REST API가 아닌 MCP(Model Context Protocol)로 제공한다. AI 에이전트가 의약품 데이터를 자연어 컨텍스트에서 활용할 수 있게 하는 것이 핵심 가치.

## 2. 토큰 효율성

모든 응답은 토큰 소비를 최소화하도록 설계한다.
- `response_filters.py`로 API 응답을 압축
- 전용 도구(same_ingredient 등)를 범용 도구보다 우선
- PageSize 최소화 원칙

## 3. 단방향 의존성

레이어 간 import는 반드시 위에서 아래로만 허용한다. 순환 참조를 원천 차단하고 각 레이어의 독립적 테스트를 보장한다.

## 4. 인증 투명성

401 에러 발생 시 자동 재인증 후 1회 재시도. 사용자(에이전트)가 인증 상태를 의식하지 않고 도구를 사용할 수 있어야 한다.

## 5. Dual Entry Point 유지

Standard MCP SDK(세밀한 제어)와 FastMCP(간편 등록) 두 진입점을 유지한다. 각각의 용도가 다르며, 동일한 비즈니스 로직(druginfo/)을 공유한다.

## 6. 도메인 코드 무결성

의약품 코드 체계(ProductCode, 주성분코드, EDI코드 등)의 구조와 의미를 정확하게 보존한다. 코드 해석 로직은 문서화하고 일관성을 유지한다.

## 7. 안전한 실패

API 응답 형식이 변경되어도 서버가 중단되지 않는다. `_safe_compact()`로 필터링 실패 시 원본 데이터를 반환하는 fallback을 보장한다.
